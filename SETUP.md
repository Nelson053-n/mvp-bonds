# MVP Bonds - Setup Guide

## Prerequisites

- Python 3.10+
- pip (Python package manager)

## Installation

### 1. Create and activate a virtual environment

```bash
cd /home/nel/mvp-bonds
python3 -m venv .venv
source .venv/bin/activate  # On Windows: .venv\Scripts\activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Set environment variable for JWT

```bash
export MVP_JWT_SECRET=your_secret_key_here
```

Or on Windows:
```cmd
set MVP_JWT_SECRET=your_secret_key_here
```

For development, you can use a simple secret:
```bash
export MVP_JWT_SECRET=dev_secret_key_12345
```

### 4. Run the application

```bash
python3 -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

The application will be available at: **http://127.0.0.1:8000**

## First Run

On first run with an empty database, the application will:

1. Create database schema (users, portfolios, portfolio_items tables)
2. Create an admin user with:
   - **Username**: `admin`
   - **Password**: `<auto-generated>` (shown in logs)

**Check the console output for the auto-generated admin password!**

## Features Implemented

### Phase 1-3: Backend Foundation ✅
- User authentication with JWT tokens
- Password hashing with bcrypt
- Multi-portfolio support
- Database schema with users, portfolios, and items tables
- Automatic admin user bootstrap on first run

### Phase 4: Frontend ✅
- Login/register screen with username and password
- Portfolio selector dropdown in topbar
- Share portfolio functionality with:
  - Unique share tokens (UUID)
  - Optional password protection
  - Public read-only view at `/share/{token}`
- All API endpoints updated to use new `/portfolios/{id}/instruments/*` schema
- Bearer token authentication on all requests
- 401 error handling with automatic logout

## Usage

### 1. Register/Login
- Go to http://127.0.0.1:8000
- Register a new user or login with admin credentials

### 2. Create Portfolios
- Use the portfolio dropdown to select/create portfolios
- Add instruments by ticker symbol

### 3. Share a Portfolio
- Click the "📤 Поделиться" (Share) button
- Optionally set a password
- Copy the share link
- Share the link with others

### 4. View Shared Portfolio
- Open the share link without authentication
- If password-protected, enter the password when prompted
- View portfolio in read-only mode

## Database

The application uses SQLite with automatic migrations. Database file will be created at:
```
app/services/storage.db
```

### Key Tables
- **users**: User accounts with password hashes
- **portfolios**: Portfolio metadata and share tokens
- **portfolio_items**: Individual instruments/bonds in each portfolio

## API Endpoints

### Authentication (Public)
- `POST /auth/register` - Register new user
- `POST /auth/login` - Login user
- `GET /auth/me` - Get current user (requires auth)

### Portfolios (Authenticated)
- `GET /portfolios` - List user's portfolios
- `POST /portfolios` - Create new portfolio
- `GET /portfolios/{id}` - Get portfolio details
- `PATCH /portfolios/{id}` - Update portfolio name
- `DELETE /portfolios/{id}` - Delete portfolio
- `POST /portfolios/{id}/share` - Create share link
- `DELETE /portfolios/{id}/share` - Revoke share link

### Instruments (Authenticated)
- `GET /portfolios/{id}/table` - Get portfolio instruments
- `POST /portfolios/{id}/instruments` - Add instrument
- `PATCH /portfolios/{id}/instruments/{item_id}` - Update instrument
- `DELETE /portfolios/{id}/instruments/{item_id}` - Delete instrument
- `PATCH /portfolios/{id}/instruments/{item_id}/coupon` - Update bond coupon
- `POST /portfolios/{id}/validate` - Validate instrument data
- `POST /portfolios/{id}/import` - Import CSV file
- `GET /portfolios/{id}/export` - Export CSV file

### Public Share (No Auth Required)
- `GET /share/{token}/table` - View shared portfolio (with optional password)

### Settings (Authenticated)
- `GET /settings/notifications` - Get notification settings
- `POST /settings/notifications` - Update notification settings
- `POST /settings/notifications/test` - Send test notification

## Troubleshooting

### "Module not found" errors
Ensure you've activated the virtual environment and installed requirements:
```bash
source .venv/bin/activate
pip install -r requirements.txt
```

### JWT_SECRET not set
The application requires `MVP_JWT_SECRET` environment variable:
```bash
export MVP_JWT_SECRET=your_secret
```

### Database locked
If you get SQLite locked errors, ensure only one instance of the app is running.

### Port already in use
Change the port:
```bash
python3 -m uvicorn app.main:app --host 127.0.0.1 --port 8001
```

## Development Notes

- The application uses FastAPI with async support
- All portfolio data is cached in-memory with periodic background refresh
- Share tokens are UUIDs for uniqueness
- Passwords are hashed with bcrypt (rounds=12)
- JWT tokens expire after 72 hours
